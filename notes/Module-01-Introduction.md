# Module 1 — Introduction to the Cloud

**Status: ✅ done** · Domain 1 (Cloud Concepts) · Full notes in the [Study hub](AWS-Cloud-Practitioner.html) → Domain 1.

## Key points
- **Cloud computing** = on-demand delivery of IT resources over the internet with **pay-as-you-go** pricing. No owning/maintaining servers.
- **6 advantages** (mnemonic *Very Easy Cloud Saves Data Globally*): trade capex→opex · economies of scale · stop guessing capacity · speed/agility · stop running data centers · go global in minutes.
- **Service models** (who manages what): **IaaS** (EC2/VPC — you manage OS+app) · **PaaS** (Beanstalk/RDS/Lambda — just your app) · **SaaS** (Gmail — just use it).
- **Deployment models** (where it runs): **Cloud** · **Hybrid** · **On-premises**.
- **Client–server**: client requests → server (e.g. EC2) responds.

> Don't confuse the **6 advantages** with the **6 Well-Architected pillars** (Module 13) — different lists, both tested.
