Networking · Module 5

VPC Connectivity Menu

A VPC is your isolated private network (CIDR like 10.0.0.0/16, regional), split into subnets (one AZ each; public = has an Internet Gateway route). The single most-tested networking topic is how traffic gets in and out.

PathWhat it isOver internet?Use it for
Internet Gateway (IGW)The VPC's door to the public internet; public subnets route 0.0.0.0/0 → IGW.Yes · publicPublic-facing apps, web servers, load balancers.
VPN (via VGW)A Site-to-Site VPN: an encrypted IPsec tunnel between on-prem and the VPC. AWS side = Virtual Private Gateway, your side = Customer Gateway.Yes · encryptedQuick, cheap hybrid link over the public internet.
Direct Connect (DX)A dedicated private physical fibre line to AWS that bypasses the public internet.No · privateConsistent low latency, high bandwidth, large/sensitive data, compliance.
Memory hook

IGW = public door · VPN = encrypted tunnel over the internet · Direct Connect = private dedicated cable, no internet.

Exam trap

Direct Connect alone is private but NOT encrypted. Need a dedicated line and encryption? Run a VPN over Direct Connect. Also know by name: Transit Gateway (hub connecting many VPCs + on-prem) and VPC Endpoints / PrivateLink (reach AWS services privately, no internet).

Networking · Module 5

Security Groups vs NACLs

Packet path inward: internet → IGW → subnet's NACL (border guard) → instance's Security Group (bouncer) → EC2.

 Security GroupNetwork ACL (NACL)
Operates atInstance level (the ENI)Subnet level
StateStateful return traffic auto-allowedStateless must allow return traffic explicitly
Rule typesAllow only everything else implicitly deniedAllow + Deny can explicitly block
EvaluationAll rules evaluated togetherNumber order — first match wins
DefaultDenies all inbound, allows all outboundDefault allows all; custom denies all until you add rules
Memory hook

Security Group = Stateful · NACL = Can say no (deny) + has no memory (stateless). Need to block a specific IP? → NACL (SGs can't deny).

DNS & Edge Delivery

Amazon Route 53 = AWS's managed DNS (names → IPs) and a domain registrar. Routes to AWS resources (ELB, CloudFront, S3) and on-prem, with health checks.

Route 53 policySends traffic by…
SimpleOne record → one resource.
WeightedA % split across endpoints (A/B testing, canary).
Latency-basedThe Region with the lowest latency for the user.
GeolocationThe user's physical location.
FailoverA backup if the primary is unhealthy (disaster recovery).
Amazon CloudFront
AWS Global Accelerator
  • CDN — caches content at edge locations near users
  • HTTP/HTTPS web content (static + dynamic)
  • Lower latency + absorbs spikes; DDoS protection with Shield
  • No caching — routes over the AWS backbone
  • For TCP/UDP apps: gaming, IoT, VoIP, APIs
  • Gives static anycast IPs + fast failover
Exam trap

CloudFront caches at the edge. Global Accelerator does not cache — it speeds up non-HTTP traffic and hands you static IPs by routing over AWS's private backbone.

Storage · Module 6

The 3 Storage Types

The master key: there are only three types of storage. Every service maps to one — learn three ideas, not twelve names.

TypeWhat it really isAnalogyAWS service
BlockRaw disk cut into blocks; OS puts a filesystem on top; attaches to one machine.A hard drive in one computerAmazon EBS
FileFiles in folders over a network protocol (NFS/SMB); many machines mount at once.A shared team network driveEFS FSx
ObjectWhole objects (data + metadata + ID) in a flat pool, via URL/API; infinite scale.A web-accessible valet lockerAmazon S3
EBS — block
EFS — file
  • Attaches to one EC2 instance
  • Scope: one AZ
  • You pick the size
  • Snapshots = point-in-time backups in S3
  • Mounted by many EC2s at once
  • Scope: regional (spans AZs)
  • Auto-scales elastically
  • FSx = the Windows / specialized variant
Memory hook

EBS = one Box · EFS = Fleet (shared by many). Instance store = ephemeral, lost on stop — don't confuse with EBS.

Storage · Module 6 · most-tested

S3 Storage Classes — One Cost Dial

All classes share the same 11-nines durability — they are not safer-vs-riskier. They differ on storage cost vs retrieval cost/speed vs availability. Two questions decide everything: how often will I read it, and how fast do I need it back?

▲ Hotter · instant · pricier storageColder · slower · cheaper storage ▼
Standard
Frequent access. Free, instant reads. Default.
Intelligent-Tiering
Unknown pattern — AWS moves objects for you. No retrieval fees.
Standard-IA
Known-infrequent, instant. Pay per retrieval. 30-day min.
One Zone-IA
Same, but 1 AZ. Cheaper; lost if AZ dies. Recreatable data.
Glacier Instant
Archive accessed ~quarterly, still in milliseconds.
Glacier Flexible
Archive; retrieval in minutes to hours.
Deep Archive
Cheapest. 12–48 hr retrieval. Long-term compliance.
The 3 confusions to keep straight

Intelligent-Tiering vs Standard-IA: unknown pattern + hands-off → Intelligent-Tiering; known-infrequent → Standard-IA.  One Zone-IA: trigger is "recreatable / fine if lost," not just "cheapest."  The 3 Glaciers differ only by retrieval speed (ms / min–hrs / hrs).

Storage · Module 6

Lifecycle & Moving Big Data

Lifecycle policy = rules that slide objects down the dial as they age, e.g. Standard → Standard-IA (30d) → Glacier (90d) → delete. Exam cue: "automatically move old data to cheaper storage" → lifecycle policy (a feature, not a class).

ServiceWhat it does
Storage GatewayHybrid bridge: on-prem apps keep their protocol, data lives in AWS, hot data cached locally. Types by interface: File (S3/FSx) · Tape (VTL → S3+Glacier) · Volume (iSCSI → S3+EBS snapshots).
AWS BackupOne central place to back up across many AWS services.
Snow FamilyPhysically ship data when the network is too slow: Snowcone (portable) → Snowball (suitcase, TB–PB) → Snowmobile (a truck, exabytes).
Memory hooks

Snow family: cone < ball < mobile = small < medium < gigantic.  Storage Gateway trigger: "keep using the on-prem app but store/back up in AWS" → Storage Gateway.
